# 产品文档

Kest 面向一类很明确的开发方式：你在终端里调试接口，在编辑器里和 AI 协同写代码，同时希望测试步骤可以被记住、复用和追踪。

和传统 API 工具相比，Kest 更强调三件事：

- `上下文连续性`：请求、变量和上下游依赖可以在一次测试流中持续传递
- `AI 可读性`：测试流使用 Markdown 编写，适合人类阅读，也适合 AI 直接生成和维护
- `终端到团队的闭环`：CLI 负责执行，Web Console 负责可视化、协作和沉淀历史
