# 产品介绍

## Kest 是什么

Kest 是面向 AI 编程时代的 API 测试平台。

如果说 `curl` 太短暂、`Postman` 太厚重，那么 Kest 的核心价值就是把 API 调试和测试过程变成一条可持续演进的工作流。

它不是单纯替代某一条命令，也不是把传统 GUI 工具搬到浏览器里，而是重新定义了以下几件事：

- 请求不再是一次性动作，而是可复用、可串联、可沉淀的测试步骤
- 测试描述不再困在表单里，而是可以用 Markdown 直接表达
- AI 不只是帮你写代码，也可以理解并维护你的测试流

## Kest 为谁而做

Kest 适合以下开发者：

- 长期在终端中调试接口和跑脚本的后端或全栈工程师
- 在 Cursor、Windsurf、Cline、Copilot 等工具里与 AI 高频协作的开发者
- 需要让接口测试在个人调试和团队协作之间自然流转的团队

如果你的日常工作是边写代码、边试接口、边让 AI 帮你改逻辑，那么 Kest 会比“命令行 + 手工复制粘贴 + 零散历史记录”的组合更顺手。

## 产品由什么组成

Kest 由两部分组成：

### Kest CLI

CLI 是执行层，强调速度、可脚本化和可组合性。你可以在终端中完成：

- 初始化项目
- 发起单次请求测试
- 捕获变量并在后续步骤中复用
- 运行完整的 Markdown 流程文件
- 在失败时调用 AI 诊断原因

### Kest Web Console

Web Console 是协作层，负责补上终端工具天然不擅长的部分：

- 可视化查看测试流和结果
- 管理团队数据和历史记录
- 共享调试上下文
- 承载后续的平台能力与 AI 分析结果

## Kest 解决了什么问题

传统 API 工具通常有两个极端：

- `curl` 足够轻，但完全无状态，每次都要重新组织上下文
- `Postman` 功能很多，但在 AI 协作和终端工作流里显得偏重

Kest 的思路是在二者之间建立一条更适合现代开发的路径：

- 像命令行一样直接
- 像文档一样可读
- 像测试系统一样可复用
- 像协作平台一样可沉淀
