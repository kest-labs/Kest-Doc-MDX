## 怎么理解这张表

- `curl` 的优势是轻，但没有上下文管理能力
- `Postman` 的优势是功能面广，但工作流偏 GUI 和账号体系
- `Hurl` 在文本化测试上很强，但不会替你记录历史，也不提供 AI 协作层
- `Kest` 的重点是把本地历史、变量链路、AI 和 Markdown 流结合在一起

因此，Kest 的差异不在于“某一个功能”，而在于它把请求、回放、诊断和测试资产管理放进了同一个终端循环里。
