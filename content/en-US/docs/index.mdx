# Product Docs

Kest is built for a specific kind of developer workflow: you debug APIs in the terminal, write code with AI copilots in your editor, and want test steps to stay readable, repeatable, and connected.

Compared with traditional API tools, Kest focuses on three things:

- `Context continuity`: requests, captured values, and downstream steps can be chained together
- `AI readability`: flows are written in Markdown, so both humans and AI agents can work with them directly
- `Terminal-to-team workflow`: the CLI executes fast, while the web console gives you visibility and collaboration
