# Product Introduction

## What Kest Is

Kest is an API testing platform built for the AI programming era.

If `curl` is too stateless and `Postman` is too heavy, Kest is the layer that turns API debugging into a workflow you can keep, extend, and share.

It is not just a shorter command or a browser copy of an older GUI tool. It changes the model in a few important ways:

- requests become reusable, chainable test steps instead of one-off commands
- tests are written in Markdown instead of being trapped in forms
- AI can read, generate, and maintain the same flow files that developers use

## Who It Is For

Kest fits developers who:

- spend most of their time testing APIs in the terminal
- work with AI copilots such as Cursor, Windsurf, Cline, or Copilot
- want API testing to move cleanly from individual debugging into shared team workflows

If your day-to-day work already mixes code changes, API probes, and AI-assisted iteration, Kest is designed for that environment.

## The Product Surfaces

Kest has two primary surfaces.

### Kest CLI

The CLI is the execution layer. It is optimized for speed, scriptability, and composability. You use it to:

- initialize a project
- run single request tests
- capture values and reuse them in later steps
- execute full Markdown flow files
- ask AI to diagnose failures with `kest why`

### Kest Web Console

The web console is the collaboration layer. It fills the gaps that terminal tools do not cover well:

- visualizing flows and results
- storing team data and history
- sharing debugging context
- hosting richer platform logic and AI-assisted analysis

## The Problem Kest Solves

Traditional API tools usually fall into two extremes:

- `curl` is fast and universal, but it forgets everything
- `Postman` is powerful, but it often feels too heavy for terminal-first AI workflows

Kest sits between those extremes:

- as direct as the command line
- as readable as documentation
- as reusable as a test system
- as collaborative as a platform
