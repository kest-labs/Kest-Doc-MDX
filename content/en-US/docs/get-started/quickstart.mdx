# Quickstart

This page keeps the first-run path intentionally short. The goal is to do three things quickly:

- install the CLI
- initialize a project
- run your first API test

## 1. Install the CLI

```bash
curl -fsSL https://kest.dev/install.sh | sh
```

The current release flow is built with GoReleaser and supports:

- macOS
- Linux
- Windows

## 2. Initialize a Project

Inside your project directory, run:

```bash
kest init
```

This creates the basic Kest project scaffolding so you can keep request history, flows, and related configuration in one place.

## 3. Send Your First Request

Test an endpoint directly:

```bash
kest get /api/users -a "status==200"
```

Here, `-a` adds an assertion. This example says:

- send `GET /api/users`
- expect the response status to be `200`

## 4. Capture Values for Later Steps

If a response contains data you need later, capture it in the CLI:

```bash
kest post /api/login -d '{"user":"admin"}' -c "token=data.token"
```

This command:

- sends a login request
- extracts `data.token` from the response body
- stores it as `token` for later use

## 5. Run a Markdown Flow

Instead of keeping a chain of requests in shell history, write them as a flow:

```bash
kest run auth.flow.md
```

This is one of the core differences in Kest. A flow can be read by humans and manipulated directly by AI tools.
